# TODO: два скилла для проверки цитирования

---

## Архитектура

Два скилла, чёткое разделение ответственности:

- **`check_citations_reliability`** — проверяет, реально ли существует статья, и генерирует карточку статьи
- **`check_citations_correctness`** — берёт карточки (из скилла 1) и проверяет, корректно ли автор на них ссылается в тексте

---

## Скилл 1 — `check_citations_reliability`

### Цель
Для каждой записи в `bibliography.bib`:
1. Проверить, реальна ли статья (не выдумана)
2. Создать подробную карточку статьи в `papers_cards/<key>.md`

### Источники данных (в порядке приоритета)
1. PDF (если есть прямая ссылка или DOI → PDF)
2. arXiv (если есть arXiv ID или title match)
3. Semantic Scholar API (зарезервировано)
4. Perplexity `sonar` — fallback

**Важно:** Claude (оркестратор) сам не читает статью и не формулирует по abstract. Весь анализ — через qwen3-235b.

### Структура карточки `papers_cards/<key>.md`

```markdown
---
key: <bibkey>
title: <полное название>
authors: <авторы>
year: <год>
venue: <журнал/конференция/arXiv>
doi: <если есть>
arxiv_id: <если есть>
pdf_url: <если найден>
semantic_scholar_id: <зарезервировано>
verified: true/false
confidence: 0–100
source_used: pdf | arxiv | perplexity | semantic_scholar
---

## Цель работы
<что авторы хотели решить>

## Закрытый гэп
<какая проблема до этой работы была открыта>

## Метрики и датасеты
<на каких датасетах, какие метрики использовались>

## Результаты
<конкретные числа и выводы авторов>

## Верификационный вердикт
REAL / FAKE / UNCERTAIN — <обоснование qwen3>
```

### Поведение скилла
- Запуск: `uv run python tools/check_papers.py` (существующий инструмент — расширить)
- Флаги: `--test` (5 записей), `--recheck key1 key2`, `--prompt "..."`, `--hints key:"..."`
- Кэш: если карточка уже есть и `verified=true` → пропустить (если не `--recheck`)
- Параллельность: все статьи асинхронно
- Логирование: все промпты/ответы qwen3 через `log_io.py`
- Итог: `papers_results.json` (ключ → verdict + confidence) + папка `papers_cards/`

## Скилл 2 — `check_citations_correctness`

### Цель
Для каждой статьи в `bibliography.bib`:
1. Взять карточку из `papers_cards/<key>.md` (результат скилла 1)
2. Найти все абзацы в .tex файлах, где встречается ключ цитирования
3. Отправить абзац + карточку в qwen3
4. qwen3 определяет: корректно ли автор ссылается на статью
5. Сформировать HTML-отчёт

**Perplexity не используется.** Источник правды — только карточки из скилла 1.

### Поведение скилла
- Запуск: `uv run python tools/check_context.py`
- Флаги: `--test` (3 статьи), `--only key1 key2`
- Параллельность: до 10 абзацев одновременно (asyncio + semaphore)
- qwen3 пишет вердикт и анализ **на русском**
- Логирование: через `log_io.py`

### Вердикты qwen3 для каждого абзаца
- `КОРРЕКТНО` — ссылка соответствует смыслу статьи
- `НЕКОРРЕКТНО` — ссылка не соответствует / вводит в заблуждение
- `ЧАСТИЧНО` — ссылка корректна, но неполна или неточна

### HTML-отчёт `citation_report.html`
- Сводная таблица: статья | кол-во упоминаний | корректных | некорректных | частичных
- Детальные таблицы: абзац | файл | строка | вердикт | анализ qwen3
- Некорректные и частичные — подсвечены цветом
